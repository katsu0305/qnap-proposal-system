## 📊 ステップ2: 容量・ベイ数の決定

### RAID構成別の実効容量計算

```
【RAID 0】（冗長性なし）
実効容量 = HDD容量 × ベイ数
例: 4TB × 4ベイ = 16TB

【RAID 1】（ミラーリング）
実効容量 = HDD容量 × (ベイ数 / 2)
例: 4TB × 4ベイ = 8TB（50%効率）

【RAID 5】（パリティ1）
実効容量 = HDD容量 × (ベイ数 - 1)
例: 4TB × 4ベイ = 12TB（75%効率）

【RAID 6】（パリティ2）
実効容量 = HDD容量 × (ベイ数 - 2)
例: 4TB × 4ベイ = 8TB（50%効率）

【RAID 10】（ミラー+ストライプ）
実効容量 = HDD容量 × (ベイ数 / 2)
例: 4TB × 4ベイ = 8TB（50%効率）
```

### 必要ベイ数の算出ロジック

```
STEP 1: 必要実効容量を算出
  必要実効容量 = 現在のデータ量 + (年間増加量 × 3年) + バッファ20%
  
  例: 現在10TB + (3TB/年 × 3年) + 3.8TB = 22.8TB

STEP 2: RAID構成を決定
  IF (業種 = 医療 OR 金融 OR ミッションクリティカル) THEN
      RAID 6（2台故障許容）
  ELSE IF (容量 > 20TB) THEN
      RAID 5 or RAID 6
  ELSE IF (ベイ数 = 2) THEN
      RAID 1
  ELSE
      RAID 5

STEP 3: 必要ベイ数を逆算
  必要ベイ数 = 必要実効容量 / (HDD容量 × RAID効率)
  
  例: RAID 6, HDD 8TB想定
    22.8TB / (8TB × 50%) = 5.7ベイ → 6ベイ必要
```

### ベイ数決定マトリクス

| 必要実効容量 | RAID 1 | RAID 5 | RAID 6 | 推奨ベイ数 |
|------------|--------|--------|--------|-----------|
| ～4TB | 2ベイ（2TB×2） | 4ベイ（2TB×4） | - | **2ベイ** |
| 4～8TB | 2ベイ（4TB×2） | 4ベイ（2TB×4） | - | **2～4ベイ** |
| 8～16TB | 4ベイ（4TB×4） | 4ベイ（4TB×4） | 4ベイ（4TB×4） | **4ベイ** |
| 16～24TB | - | 4ベイ（8TB×4） | 4ベイ（8TB×4） | **4ベイ** |
| 24～40TB | - | 4ベイ（12TB×4） | 6ベイ（8TB×6） | **4～6ベイ** |
| 40～80TB | - | - | 12ベイ（8TB×12） | **12ベイ** |
| 80TB～ | - | - | 12ベイ（12TB～×12） | **12ベイ** |

## 📊 ステップ5: HDDグレード・容量の決定

### HDDグレード選定ロジック

```
IF (用途 = 監視カメラ録画) THEN
    → WD Purple（監視カメラ専用）
    
ELSE IF (業種 = 医療 OR 官公庁 OR 長期保存) THEN
    → ニアライン（5年保証、高信頼性）
    
ELSE IF (用途 = 24時間稼働 OR エンタープライズ) THEN
    → エンタープライズ（最高信頼性）
    
ELSE IF (用途 = 超高速 OR 動画編集 OR 仮想化) THEN
    → MDSSD or HISSD（オールフラッシュ）
    
ELSE
    → WD Red（NAS標準、コスパ良好）
```

### HDDグレード比較表

| グレード | 保証期間 | MTBF | 用途 | 価格帯 | エレコム取扱い |
|---------|---------|------|------|--------|--------------|
| **WD Red** | 3年 | 100万時間 | 一般NAS用途 | 標準 | ⭕ |
| **WD Purple** | 3年 | 180万時間 | 監視カメラ録画 | 標準 | ⭕ |
| **ニアライン** | 5年 | 200万時間 | 長期保存、医療 | +20～30% | ⭕ |
| **エンタープライズ** | 5年 | 250万時間 | 24時間稼働 | +40～60% | ⭕ |
| **MDSSD** | 3～5年 | - | 超高速 | +200～300% | ⭕ |
| **HISSD** | 3～5年 | - | 高速 | +150～250% | ⭕ |

### HDD容量の選定

```
STEP 1: 1台あたりのHDD容量を決定
  1台容量 = 必要実効容量 / (ベイ数 × RAID効率)
  
  例: 24TB必要、4ベイ、RAID 6
    24TB / (4 × 0.5) = 12TB/台

STEP 2: 標準容量に切り上げ
  標準容量: 1TB, 2TB, 3TB, 4TB, 6TB, 8TB, 10TB, 12TB, 14TB, 16TB, 18TB, 20TB, 22TB, 24TB, 26TB
  
  12TB → 12TB HDD × 4台

STEP 3: 価格と容量のバランス確認
  TB単価が最も安い容量帯: 8TB～12TB
  大容量ほどTB単価は安いが、故障時の影響大
```

## 📊 ステップ6: 保守レベルの決定

### 保守レベル選定ロジック

```
IF (業種 = 医療 OR 金融 OR 製造24時間稼働) THEN
    IF (ダウンタイム許容 < 4時間) THEN
        → 24時間365日オンサイト保守
    ELSE IF (ダウンタイム許容 < 1日) THEN
        → オンサイト保守
    ELSE
        → センドバック保守（先出し）
        
ELSE IF (業種 = 官公庁 OR 学校 OR 中規模企業) THEN
    IF (予算あり) THEN
        → センドバック保守 or オンサイト保守
    ELSE
        → 延長保証（2～3年）
        
ELSE
    → 標準保証（2～3年）
```

### 保守レベル比較表

| 保守レベル | 対応時間 | 復旧目標 | 価格目安（年額） | 推奨業種 |
|-----------|---------|---------|----------------|---------|
| **標準保証** | 営業時間内 | 1週間 | 無料 | 小規模オフィス |
| **延長保証（1～3年）** | 営業時間内 | 1週間 | 本体価格の5～15% | 一般企業 |
| **センドバック保守** | 24時間受付 | 1～2営業日 | 本体価格の10～30% | 中規模企業 |
| **オンサイト保守** | 営業時間内 | 当日～翌営業日 | 本体価格の20～40% | 大規模企業、学校 |
| **24h365dオンサイト** | 24時間365日 | 4時間以内 | 本体価格の40～80% | 医療、金融、製造 |

```
□ 容量は3年後まで十分か？（年間増加量 × 3年 + バッファ20%）
□ ユーザー数に対して性能は十分か？
□ ネットワークインフラは対応しているか？（10GbE導入時）
□ バックアップ先は確保されているか？
□ RAID構成は業種・重要度に適しているか？
□ 保守レベルはダウンタイム許容度に合っているか？
□ 予算内に収まっているか？
□ 拡張性は確保されているか？（メモリ増設、HDD交換等）
□ 電源容量・UPS容量は十分か？
□ 設置スペース・ラックは確保されているか？
```

### よくある選定ミス

| ミス | 正しい対応 |
|------|-----------|
| ❌ 現在の容量だけで選定 | ⭕ 3年後の容量で選定 |
| ❌ RAID効率を考慮していない | ⭕ RAID 6なら50%効率 |
| ❌ バックアップ先を忘れている | ⭕ 別NASまたはクラウド確保 |
| ❌ ネットワーク機器の対応確認不足 | ⭕ スイッチ・ケーブルも10GbE対応 |
| ❌ 保守契約なし | ⭕ ミッションクリティカルならオンサイト |
| ❌ 一番安い構成を選択 | ⭕ TCO（5年総コスト）で比較 |
