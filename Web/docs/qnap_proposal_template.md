## 2.2 現状の問題点

### 問題1: 容量不足

```
【現象】
・空き容量が残り10%（400GB）
・毎月約50GB増加
・あと8ヶ月で満杯になる見込み

【影響】
・新規データの保存ができなくなる
・業務に支障が出る
・外付けHDDへの分散保存でセキュリティリスク
```

### 問題2: 速度の遅さ

```
【現象】
・大容量ファイル（10GB）の転送に約90秒
・複数ユーザー同時アクセス時に遅延
・CADファイル保存時に待ち時間発生

【影響】
・業務効率の低下
・社員のストレス増加
・時間的損失（1日あたり累計30分程度）
```

### 問題3: 信頼性の不安

```
【現象】
・導入から5年経過（メーカー保証切れ）
・HDDの経年劣化の懸念
・RAID 1構成（1台故障で危険）

【影響】
・データ消失のリスク
・障害時の復旧に1週間以上
・業務停止による損失
```

### 問題4: 拡張性の限界

```
【現象】
・2ベイ構成（拡張不可）
・容量増強にはNAS買い替えが必須
・年々増えるデータ量に対応できない

【影響】
・頻繁な買い替えが必要
・データ移行の手間とコスト
・中長期的な計画が立てられない
```

【発生確率】: 中～高

5年経過したHDDの故障率は年間約8%。
RAID 1構成では1台故障時にデータ消失の危険。
バックアップ体制も不十分で、復旧不可能な可能性。

```
【発生確率】: 中
【影響度】: 大

20名の業務が停止。1日あたりの損失は
最低でも50万円と見積もられる。

```

```

【影響度】: 大

## 3.3 主要機能のご紹介

### 機能1: 高速ファイル転送

```
【2.5GbE ネットワーク】
従来の1GbEの2.5倍の帯域幅を実現。

転送速度比較:
┌──────────────────────────┐
│ 10GBファイルの転送時間      │
├──────────────────────────┤
│ 現在（1GbE）: 約90秒        │
│ 導入後（2.5GbE）: 約36秒    │
│                             │
│ 改善効果: 2.5倍高速化       │
└──────────────────────────┘

【Link Aggregation】
2つの2.5GbEポートを結合し、
最大5Gbps（理論値）の転送速度を実現。
```

### 機能2: 高い信頼性

```
【RAID 5構成】
データを3台のHDDに分散保存し、
1台のHDDにパリティ情報を記録。
1台のHDD故障時もデータ保護。

HDD構成:
┌────┬────┬────┬────┐
│HDD1│HDD2│HDD3│HDD4│
├────┼────┼────┼────┤
│データ│データ│データ│パリティ│
└────┴────┴────┴────┘
↑ どれか1台が故障してもOK

【ホットスワップ】
電源を入れたまま故障HDDを交換可能。
業務を止めずにメンテナンス実施。

【スナップショット】
ファイルの変更履歴を自動保存。
誤削除やランサムウェア攻撃からデータを保護。
```

### 機能3: 簡単な管理

```
【直感的なWeb管理画面】
ブラウザから簡単にNASを管理。
専門知識不要で設定変更が可能。

主な管理機能:
✓ ユーザー・グループ管理
✓ 共有フォルダ設定
✓ アクセス権限設定
✓ バックアップ設定
✓ 容量監視
✓ アラート通知

【モバイルアプリ】
スマートフォンから
・ファイルアクセス
・システム状態確認
・アラート受信
が可能。
```

### 機能4: 柔軟なバックアップ

```
【多彩なバックアップ方法】

1. ローカルバックアップ
   └→ USB接続外付けHDDへ自動バックアップ

2. リモートバックアップ
   └→ 他拠点のNASへネットワーク経由でバックアップ

3. クラウドバックアップ
   └→ Amazon S3、Google Drive等へバックアップ

4. スナップショット
   └→ NAS内でファイルの履歴を保存（世代管理）

【スケジュール設定】
毎日・毎週・毎月など、

### Q1: データ移行はどのように行いますか？

```
A: 以下の手順で安全にデータ移行を行います。

1. 既存NASのデータをそのまま保持
2. 新NASへデータをコピー（夜間・休日実施）
3. ファイル数・容量を照合して確認
4. ユーザーによる動作確認
5. 問題なければ既存NASから新NASへ切り替え
6. 既存NASは1ヶ月間保持（ロールバック用）

データ移行中も既存NASは通常通りご利用いただけます。
```

### Q2: 現在のネットワークで2.5GbEは使えますか？

```
A: 以下の条件を満たせば2.5GbEをご利用いただけます。

【必要な機器】
✓ 2.5GbE対応スイッチ
✓ 2.5GbE対応LANケーブル（Cat5e以上）
✓ 2.5GbE対応PCのNIC

既存のスイッチが1GbEの場合、1GbEで動作します。
将来的に2.5GbEスイッチに交換すれば、
NASの設定変更なしで2.5GbE速度を活用できます。
```

### Q3: RAID 5とRAID 6の違いは？

```
A: 冗長性と容量のバランスが異なります。

【RAID 5】
・HDD: 4台構成の場合
・実効容量: 3台分（75%）
・冗長性: 1台まで故障OK
・速度: 速い

【RAID 6】
・HDD: 4台構成の場合
・実効容量: 2台分（50%）
・冗長性: 2台まで故障OK
・速度: やや遅い

貴社の場合、4ベイ構成なので、容量と
信頼性のバランスが良いRAID 5を推奨します。
```

### Q4: 障害が発生した場合の対応は？

```
A: センドバック保守に加入の場合、以下の流れです。

【HDD故障の場合】
1. NASが自動でアラート通知
2. サポートセンターに連絡
3. 24時間以内に交換HDDが到着
4. 電源を入れたままHDDを交換
5. 自動でRAID再構築開始
6. 故障HDDは返送（着払い）

【NAS本体故障の場合】
1. サポートセンターに連絡
2. 24時間以内に代替機が到着
3. HDDを故障機から代替機へ移植
4. 即座に復旧
5. 故障機は返送（着払い）

データは保護され、業務停止時間を最小化できます。
```
