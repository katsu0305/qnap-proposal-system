## 🎯 ヒアリング項目の優先順位と理由

### 最重要項目（必ず聞く）

| 項目 | 理由 | 選定への影響 |
|------|------|------------|
| **1. 業種・用途** | 業種によって最適なモデル・構成が異なる | ・医療: ランサムウェア対策必須<br>・建設: 大容量図面データ<br>・製造: 24時間稼働要件 |
| **2. 利用目的** | 単なるファイル共有か、高度な機能が必要か判断 | ・バックアップ専用 → ニアラインHDD<br>・監視カメラ → NVR対応機種<br>・仮想化 → メモリ大容量 |
| **3. 容量** | ベイ数・HDD容量の決定に直結 | ・5TB未満 → 2ベイ<br>・20TB以上 → 4ベイ以上<br>・100TB以上 → 12ベイ |
| **4. ユーザー数** | CPU性能・メモリ容量・ネットワーク速度を決定 | ・5人以下 → 2GB RAM, 1GbE<br>・50人以上 → 4GB RAM, 2.5GbE以上 |
| **10. 予算感** | 製品ラインの絞り込み | ・コスト優先 → TS-233シリーズ<br>・高性能 → TS-H1277AFXシリーズ |

### 重要項目（製品絞り込み後に確認）

| 項目 | 理由 | 選定への影響 |
|------|------|------------|
| **5. 設置形態** | 設置場所の制約確認 | ・ラック搭載 → ラックマウント型<br>・デスク設置 → タワー型 |
| **7. ネットワーク速度** | パフォーマンス要件の確認 | ・動画編集 → 10GbE必須<br>・一般オフィス → 1GbE十分 |
| **9. 保守要件** | ランニングコスト・ダウンタイム許容度 | ・24h365d → オンサイト保守<br>・通常業務 → 標準保証 |

### 詳細設計時項目（契約前の最終確認）

- 項目11～15は、機種選定後の詳細構成決定時に確認
- HDDグレード、バックアップ方式、アクセス方法などの具体化

## ✅ ヒアリング時の注意点

### 顧客が見落としがちなポイント

1. **将来的な容量増加**
   - 現在の容量だけでなく、3～5年後の増加予測を確認
   - 例: 「現在5TBですが、年間2TB増加予定」→ 4ベイ以上推奨

2. **バックアップ先の確保**
   - NAS導入 ≠ バックアップ完了
   - 「NAS自体のバックアップはどうするか？」を必ず確認

3. **ネットワークインフラの確認**
   - 10GbE NASを導入しても、スイッチが1GbEでは意味がない
   - 既存ネットワーク機器の確認必須

4. **ランサムウェア対策の重要性**
   - 特に医療・金融・製造業では必須
   - オフラインバックアップ、スナップショット機能の説明

5. **保守契約の必要性**
   - ダウンタイムによる損失を試算
   - 「1日業務停止したらいくら損失？」と逆算

### ヒアリング時のトーク例

```
【容量確認】
❌ 悪い例: 「何TBのNASが必要ですか?」
⭕ 良い例: 「現在お使いのストレージは何TB埋まっていますか？
          また、年間どのくらいデータが増えていますか？」

【用途確認】
❌ 悪い例: 「NASで何をしたいですか？」
⭕ 良い例: 「ファイル共有以外に、バックアップや監視カメラ録画、
          テレワーク時の外部アクセスなど、検討されていますか？」

【保守確認】
❌ 悪い例: 「保守契約は必要ですか？」
⭕ 良い例: 「もしNASが故障した場合、業務が何日止まっても大丈夫ですか？
          1日あたりの損失額はどのくらいでしょうか？」
```
